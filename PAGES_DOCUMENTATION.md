## 🔍 Механізм парсингу вакансій

### Як працює парсер

**Етап 1: HTML Scraping (швидкий)**
- Використовує `cheerio` для парсингу HTML
- Шукає текст у всіх тегах `<a>` на сторінці
- Перевіряє наявність ключових слів у тексті посилань

**Етап 2: AI Fallback (якщо HTML не знайшов нічого)**
- Запускається `scrapeCompanyWithAI()` з `lib/ai-scraper.ts`
- Використовує Playwright для відкриття справжнього браузера
- Чекає завантаження JavaScript (2 сек)
- Збирає весь текст та посилання зі сторінки
- Відправляє в GPT-4o-mini для аналізу
- GPT повертає JSON з вакансіями

**Вартість:**
- HTML scraping: безкоштовно
- AI scraping: ~$0.001-0.005 за компанію (тільки коли HTML не знайшов нічого)

### Ключові слова для пошуку

**Включити (INCLUDE):**
- Project Manager
- Delivery Manager
- Program Manager
- Project Lead
- Scrum Master
- PM (з варіаціями: " pm ", " pm.", ", pm" тощо)

**Виключити (EXCLUDE):**
- Junior (будь-які варіанти)
- Product (Product Manager, Product Owner тощо)

### Збереження результатів
- Вакансії зберігаються в таблиці `jobs` в Supabase
- Використовується `upsert` з `onConflict: "url"` для уникнення дублікатів
- Автоматично створюється запис з полями:
  - `company_id`, `company_name`, `title`, `url`, `description`, `experience`, `is_relevant`

## 🔐 Система авторизації

**NextAuth.js з Google Provider:**

1. Користувач клацає "Увійти через Google"
2. Редирект на Google OAuth
3. Після успішного входу:
   - Створюється сесія в NextAuth
   - Callback `signIn` в `lib/auth.ts` автоматично створює користувача в Supabase
   - Редирект на `/dashboard`

**Захищені маршрути:**
- `/dashboard/*` - всі сторінки вимагають авторизації
- Перевірка: `useSession()` hook з NextAuth
- Якщо не авторизований → редирект на `/auth/signin`

**Вихід:**
- Кнопка "Вийти" в хедері
- Виклик `signOut({ callbackUrl: "/" })`
- Знищення сесії та редирект на головну
